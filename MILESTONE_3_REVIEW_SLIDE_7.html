Slide 7 · Config-loader integration

What implementation should smell like

Expected code shape

type HostConfig struct {
    Profiles []string `toml:"profiles" config:"profile" help:"profile to apply; repeatable or comma-separated"`
    Sources  []HostSource `toml:"source"`
}

envLoader, _ := configloader.NewEnvironmentLoader[HostConfig](
    "cubby",
    configloader.OSEnv(),
)

flagLoader, _ := pflagloader.NewLoader[HostConfig](cmd.Flags())

host, _, err := configloader.Load(
    defaults,
    fileLoader,  // host .cubby.toml
    envLoader,   // CUBBY_PROFILE
    flagLoader,  // --profile
)
defaults → host file → env → flags

Review checks

  • --profile is registered before Cobra parses args.
  • Only profile-scoped commands overlay env/flags onto host profiles.
  • Source configs still load from each source repo after host paths resolve.
  • CSV, repeated flags, trimming, and dedupe come from config-loader slice semantics.
  • --profile= is tested as an explicitly changed empty flag.

Red flag: if link/unlink still only call cmd.Flags().GetStringSlice("profile"), host defaults and env fallback are probably not implemented correctly.
