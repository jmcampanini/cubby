Config loading

Cubby can load a real host plus registered sources

Host config

[[source]]
name = "work"
path = "~/Code/work-dotfiles"
profiles = ["work"]
ignore_conflicts = false

Source config

profiles = ["work"]

ignore = [
  "**/*.draft.*",
]

Configs are parsed through github.com/jmcampanini/go-config-loader behind internal/config.

Commands work from nested directories

→

find .cubby.toml
→
Use root

Source paths support absolute paths, ~/ expansion, and relative paths resolved from the discovered host root.

Profile model

Required gitignore patterns come from source declarations

Milestone 1 computes the sorted union of profiles declared by registered source repos.

source A: profiles = ["work", "client"]
source B: profiles = ["work", "home"]

union: client, home, work

Importantly, gitignore check/sync use source-declared profiles, not host source.profiles selections.

Gitignore guard

The first real end-to-end behavior

Compute

For each profile:

*.work.*
*.work

Check

Read host .gitignore, treating missing files as empty.

Sync

Append missing patterns once, with readable newline handling.

Script-friendly output

Missing patterns are printed plainly

$ cubby gitignore check
*.work.*
*.work

$ echo $?
1

$ cubby gitignore sync
*.work.*
*.work

$ cubby gitignore check
$ echo $?
0

Errors go to stderr through the normal command path; no JSON or fancy terminal UI is required for Milestone 1.

Validation

Obvious config problems fail clearly

Missing host config

No .cubby.toml found from cwd upward.

Missing source path

Registered source path does not exist.

Missing source config

Source repo lacks cubby.toml.

No profiles

Source config declares no usable profiles.
